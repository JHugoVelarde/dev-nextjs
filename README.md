# 🚀 Ruta de Aprendizaje de Next.js

Para aprender **Next.js** de manera efectiva y no frustrarte en el camino, no necesitas ser un experto absoluto, pero sí tener unos cimientos sólidos en el ecosistema de JavaScript moderno. Next.js es un marco de trabajo (*framework*) construido sobre React, por lo que intentar aprenderlo sin saber React es como querer decorar un pastel antes de saber hornearlo.

Aquí tienes la hoja de ruta esencial:

### 1. Requisitos Previos (Fundamentos)
Antes de tocar Next.js, asegúrate de sentirte cómodo con:
* **JavaScript Moderno (ES6+):** Debes dominar las funciones de flecha (`=>`), desestructuración de objetos, *spread operator*, promesas y `async/await`.
* **Fundamentos de React:** Es vital entender qué son los **Componentes**, el uso de **Props**, y los Hooks básicos como `useState` y `useEffect`.
* **HTML y CSS:** Conceptos de maquetación y cómo funciona el DOM.

---

### 2. Conceptos Clave de Next.js
Una vez que domines lo anterior, Next.js introduce conceptos específicos que lo hacen potente:

| Concepto | Qué significa |
| :--- | :--- |
| **Server Components** | Componentes que se renderizan en el servidor para mejorar la velocidad. |
| **File-based Routing** | Crear rutas creando archivos en carpetas (ej: `app/contacto/page.js` crea la URL `/contacto`). |
| **Data Fetching** | Cómo traer datos de una API (SSR, SSG e ISR). |
| **Optimización** | El uso de componentes nativos como `<Image />` y `<Link />` para rendimiento. |

---

### 3. Herramientas de Desarrollo
Para empezar a picar código, necesitarás tener instalado lo siguiente:
1.  **Node.js (Versión LTS):** Es el motor que ejecuta JavaScript en tu computadora.
2.  **Terminal/Consola:** Para ejecutar comandos.
3.  **Editor de Código:** Visual Studio Code es el estándar de la industria.
4.  **Gestor de paquetes:** `npm`, `yarn` o `pnpm` (vienen con Node.js).

---

### 4. ¿Por dónde empezar?
La mejor forma de iniciar hoy es utilizando el **App Router**, que es el estándar actual de Next.js. Puedes crear tu primer proyecto ejecutando este comando en tu terminal:

`npx create-next-app@latest`

Este comando te hará preguntas sobre si quieres usar **TypeScript**, **Tailwind CSS** o **ESLint**. Mi consejo: si estás empezando, acepta Tailwind CSS para el diseño, pero mantén las cosas simples.
